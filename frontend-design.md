# AI陪练平台 · 前端界面设计文档

> 版本：v1.0 | 状态：已 Opus 审核确认
> 用途：提供给 claude-design 出稿 UI 视觉，再交给开发按此实现
> 覆盖范围：学习端（L01-L11）+ 管理后台（A01-A15, SA01-SA03）

---

## 一、设计哲学与核心原则

### 1.1 双应用设计基调

**学习端（Learning App）**：Apple iOS 家族视觉语言。清透、专注、有温度。让被培训的员工感受到的不是一个冷硬的考试系统，而是一个贴心的一对一辅导伙伴。

**管理后台（Admin App）**：Apple macOS 视觉语言。高效、专业、信息密度适中。让管理员操作时感受到精良的 Mac 原生 App 质感——一切都在它该在的地方。

### 1.2 五项设计原则

1. **呼吸感优先**：大量留白，内容不拥挤，卡片之间有足够间距
2. **毛玻璃分层**：`backdrop-filter: blur()` 区分视觉层级（导航栏/卡片/浮层），创造空间纵深感
3. **状态反馈清晰**：任务进度、阅读完成状态、录音/AI说话状态，每一个状态都有明确视觉语言
4. **语音交互仪式感**：录音和 AI 对话时的 UI 要有足够存在感（动态波形、脉冲光环、状态环），让用户感受到"这是一次真实的对话"
5. **移动优先，PC 增强**：核心流程在手机上完整可用，PC 端在同等布局基础上展示更多辅助信息（侧栏、双列）

```

## 三、关键共用组件规格

### 3.1 AudioRecorder（录音组件）

这是整个产品最核心的交互组件，出现在知识问答和模拟对练中，必须精心设计。

**两种触发模式**（顶部可切换）：
- **按住说话**（Push-to-Talk，默认，适合手机）：长按按钮开始录音，松开结束
- **点击开始/停止**（Toggle，适合 PC 鼠标操作）：点击开始，再点击停止

**状态与视觉**：

```
[空闲态]
  大圆形按钮（Primary蓝），中心麦克风图标
  按钮下方文字：「按住说话」

[录音中]
  按钮变为深红色 #FF3B30
  按钮四周：实时音量驱动的波形环（SVG路径动画，基于 Web Audio API AnalyserNode）
  按钮中心：录音时长计时器（00:00 格式，超过 2 分钟自动停止）
  按钮下方文字：「录音中，松手结束」
  屏幕边缘：红色脉冲光晕（营造录音中的存在感）

[录音完成 - 等待提交]
  按钮变回蓝色，图标变为播放按钮
  按钮上方：音频波形回放条（可点击播放重听）
  下方出现两个按钮：[重新录音] [提交答案]

[处理中]
  按钮变为浅灰色，禁用状态
  显示「AI 正在评分...」加载动画（三点脉冲）
```

**iOS Safari 处理**：
- 录音格式：`audio/mp4;codecs=mp4a.40.2`（iOS），降级 `audio/webm;codecs=opus`（Chrome）
- 长按需要 `addEventListener('contextmenu', e => e.preventDefault())` 阻止系统菜单
- `getUserMedia` 必须在用户手势事件回调中调用，不能自动触发

### 3.2 PDFReader（PDF阅读组件）

基于 `pdf.js`，封装为 Vue 组件，用于材料阅读页。

**功能**：
- 分页展示（每页铺满容器宽度，高度自适应）
- 上下翻页按钮（手机支持滑动手势）
- 顶部页码显示：`当前页 / 总页数`
- 每次翻页 → 调用 `emit('page-change', {page, total})` → 父组件上报后端
- 缩放控制（+/-，范围 50%-200%）
- 加载进度显示

**已读状态判定**：
- 后端维护 `visited_pages[]` 数组
- 必须翻过每一页才标记该材料为"已读"（防止跳页）
- 前端不做本地判定，以后端 `is_completed` 为准

### 3.3 VoiceChatBubble（对话气泡组件）

用于模拟对练对话界面（L10）的消息展示。

**AI 气泡（左对齐）**：
- 左侧：虚拟角色圆形头像（AI 说话时有蓝色脉冲光环）
- 气泡背景：白色，左侧微阴影
- 文字内容（打字机效果，逐字显示）
- 右下角：播放按钮（▶），可重播该轮 TTS 音频
- AI 处理中状态：气泡内显示三点动画（`···`）

**用户气泡（右对齐）**：
- 气泡背景：Primary 蓝色
- 文字颜色：白色
- 右下角：播放按钮（可重播自己的录音）
- 转写文字下方（practice模式结果页）：小字显示得分

**特殊状态气泡**：
- 系统提示（居中）：灰色小字（如「对话已结束，正在生成评分...」）

### 3.4 ScoreDisplay（分数展示组件）

用于知识问答结果页（L08）和模拟对练结果页（L11）。

**主分数区**：
- 数字从 0 滚动到目标分数（`countUp.js` 或手写 RAF 动画，800ms 完成）
- 分数下方：等级标签（彩色胶囊形）
  - 优秀（≥excellent_score）：绿色 `#34C759` 背景
  - 合格（≥pass_score）：蓝色 `#007AFF` 背景
  - 待提升（< pass_score）：橙色 `#FF9500` 背景（避免红色，减少打击感）

    键盘类型：tel
  
  密码输入框：
    右侧 👁 眼睛图标（切换明文）

**交互细节**：
- 手机键盘弹起时，表单区整体上移，不被遮挡
- 登录成功：页面向左滑出，主页从右滑入
- 登录失败：输入框抖动动画（shake keyframe）+ 红色提示文字

### L10 · 模拟对练 · 对话界面（设计重点）

**设计意图**：这是整个产品最有差异化的界面。用户需要感受到这是一次真实的商务对话，而非机械的问答。虚拟角色需要有"存在感"，用户的录音状态需要有"仪式感"。

```

│                                         │
│ [角色头像]  您好，我是李明华，今天...    │  ← AI气泡（左对齐）
│  ○ 光环     ▶ 播放                      │

│                                         │
│ [角色头像]  嗯，你说的这个数据是        │
│  ●●● 动画   基于哪个行业的...           │  ← AI 思考中（三点动画）
│                                         │
│                                         │
├─────────────────────────────────────────┤
│ AI 说话中...                            │  ← AI 说话时此区域显示状态

│ └───────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

**状态机驱动 UI（5 个状态）**：

| 状态 | UI 表现 |
|---|---|
| `AI_SPEAKING` | 角色头像有蓝色脉冲光环（keyframe 动画），气泡文字打字机出现，录音按钮 disabled（灰色） |
| `USER_TURN` | 底部区域：绿色提示「轮到你了」，录音按钮高亮蓝色，轻微上下浮动动画提示 |
| `USER_RECORDING` | 录音按钮变红，外圈音量波形环（实时），计时器，「录音中...」文字 |
| `PROCESSING` | 录音按钮变灰 disabled，底部文字依次变化：「上传中...」→「识别中...」→「AI 思考中...」 |
| `SESSION_ENDED` | AI 说结束语，底部录音区淡出，浮现「查看评分结果」大按钮（弹性进入） |

**AI 角色「说话中」视觉设计（关键）**：
- 对话列表顶部固定一个「当前说话者区」（高度约 80px）
- AI 说话时：角色头像 56px 圆形，外圈 3 层同心圆脉冲光环（蓝色，不同频率，CSS animation）
- 配合波形动画（简化版，5条竖线，高度随机变化，营造真实声音感）
- AI 说完后：光环消失，区域淡出

**布局（PC，1024px+）**：

│ 任务信息面板 │            对话区域                    │

│             ├───────────────────────────────────────┤
│             │    [录音控制区，固定底部]               │
└─────────────┴───────────────────────────────────────┘
```

## 六、交互规范补充

### 6.1 Loading 状态规范

| 场景 | Loading 形式 |
|---|---|
| 页面首次加载 | 骨架屏（Skeleton），与实际布局结构一致 |
| 数据刷新/搜索 | 内容区 overlay 半透明 + 中心 spinner |
| 按钮操作 | 按钮内 spinner 替换文字，禁用按钮避免重复提交 |
| AI 评分等待 | 卡片翻转动画（rotateY 90°）+ 「AI 正在评分...」+ 三点动画 |
| AI 生成长任务 | 线性进度条（模拟进度）+ 进度描述文字 |

### 6.2 空状态规范

所有列表页在无数据时显示空状态：
- 居中插画图标（灰色线性图标，非填充）
- 主文字：「暂无 XX」
- 副文字：操作引导（「点击右上角「新增」开始添加」）
- 部分场景提供快捷操作按钮

### 6.3 Toast/通知规范

- 成功提示：绿色，左侧 ✓，顶部居中，2秒自动消失
- 错误提示：红色，左侧 ✕，顶部居中，需手动关闭
- 加载提示：蓝色 spinner，不自动消失，操作完成后替换为成功/失败
- 位置：页面顶部居中（手机）/ 右上角（PC）

### 6.4 表单校验规范

- 实时校验：输入框失焦时触发校验（不在输入中触发，减少打扰）
- 错误态：边框变红（--color-danger），输入框下方红色小字说明
- 提交时：整体校验，第一个错误字段自动滚动到可视区并 focus
- 必填字段：标签右侧 `*` 红色标记

### 6.5 确认对话框规范（危险操作）

删除、暂停企业、取消任务等危险操作，必须弹出确认 Modal：

```
[操作名称]                              ✕

确定要删除「产品知识题库」吗？
此操作不可撤销，关联任务的题库配置将失效。

                     [取消]  [确认删除]（红色）
```
